Gerenciamento de Configuração de Software

Controle de Versão

O controle de versão é uma prática, que reforça a organização, manutenção e entendimento do processo do código. Ou seja, o controle de versão é uma prática que surgiu para que seja possível visualizar as mudanças que foram feitas nos códigos e arquivos. Essa prática facilitou o processo de manutenção e correção de bugs, pois era possível visualizar e voltar para os códigos de versões anteriores, que estariam sem os bugs. Outro ponto importante referente ao controle de versão é que ele possibilita o trabalho em equipe para o mesmo projeto, sem que os desenvolvedores façam os mesmos códigos, e que nenhum código atrapalhe o código do outro.

Ferramentas mais utilizadas:

1. Git (GitHub e GitLab)

2. Subversion (SVN)

Controle de Mudanças

O controle de mudanças está relacionado a uma prática de registrar todas e quaisquer mudanças que forem feitas em códigos, documentos, e entre outras coisas, elas são registradas, e passam por um processo de avaliação da mudança, se for aprovada, são colocadas em vigor. Essa prática ajuda principalmente grandes equipes, para estarem por dentro das mudanças, reduzindo margens de erros.

Integração Contínua

Integração contínua consiste na integração entre os participantes da equipe do desenvolvimento do software, de forma, que todos os dias, realizam as mudanças no código e realizam os testes. Sempre que uma mudança é feita, o código é testado de forma automática para garantir o funcionamento. Dessa forma, garante uma menor taxa de erros. Uma das principais funções da integração contínua é melhorar a qualidade do software e ajudar na integração entre a equipe de DEV's.
